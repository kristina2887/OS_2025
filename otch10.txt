Особенности обмена информацией через FIFO для неродственных прцессов

В случае, когда информация передается от одного процесса к другому при отсутсвии между ними родственных связей, необходимо, чтобы обеспечивалось одновременное существование в системе процесса-отправителя  и процесса-получателя.
когда исполняется толкьо процесс отправитель, при попытке записать в FIFO при отсутсвии процессов, готовых читать эту информацию, процесс-отправитель блокируется до тех пор, пока какой-нибудь процесс не "заберет" эту информацию с FIFO

martynova@alexandr-Aspire-Z5751:~/OS_2025$ ./pr010_1.out
Процесс-отправитель начал работу
...

Для того, чтобы прочитать информацию, необходимо в системе запустить читающий процесс (например, с другого терминала).
После чтения записывающий процесс разблокируется и завершит работу.

Если будет запущен сперва процесс-приемник, то он также заблокируется при отсутсвие процесса в канале.
Для разблкирования процесса необходимо запустить в другом окне процесс-отправитель.

martynova@alexandr-Aspire-Z5751:~/OS_2025$ ./pr010_2.out
Процесс-приемник начал работу
...

Прочитанная строка: Строка для чтения процессом-получателем
Процесс-приемник завершил работу.
